Overview

Welcome to the Team Resistance programming sub-team! These challenges are designed to help new programmers build skills progressively while learning the FRC programming framework. Each challenge should be completed with an experienced programmer as a mentor/pair programmer.

Prerequisites:
  • WPILib installed (2025 version)
  • VS Code with WPILib extensions
  • Basic understanding of Java
  • Access to team repositories at https://github.com/teamresistance

---

Challenge 1: Simple Motor Control

Difficulty: ‚≠ê Beginner

Objective

Create a simple program to control a single NEO motor using button inputs from a joystick.

Requirements

1. Initialize a NEO motor (using REV SparkMAX controller) 2. Read input from an Xbox controller 3. Run the motor forward when Button A is pressed (50% speed) 4. Run the motor backward when Button B is pressed (50% speed) 5. Stop the motor when no buttons are pressed 6. Add safety limits: motor should not exceed ¬±0.75 throttle

Hardware Setup

  • One NEO motor with SparkMAX controller (CAN ID: 10)
  • Xbox controller connected to Driver Station

Learning Goals

  • Understanding motor controller initialization
  • Reading joystick/controller inputs
  • Basic motor control commands
  • Safety considerations in robot code

Success Criteria

  • Motor responds correctly to button inputs
  • Motor stops when buttons are released
  • Code follows team style guidelines
  • No compiler warnings

Challenge 2: Sensor Feedback & SmartDashboard

Difficulty: ‚≠ê‚≠ê Beginner-Intermediate

Objective

Extend Challenge 1 to display sensor data on SmartDashboard and add encoder-based position tracking.

Requirements

1. Use the NEO's integrated encoder to track position 2. Display the following on SmartDashboard: - Current motor speed (RPM) - Current position (rotations) - Motor temperature - Applied voltage 3. Add a "Zero Encoder" button (Button X) to reset position to 0 4. Display whether motor is running forward, backward, or stopped

Additional Features

  • Color-code the motor status display (Green = stopped, Yellow = running)
  • Log encoder position every 1 second to console
  • Add a counter showing total runtime in seconds

Learning Goals

  • Working with encoders and sensor data
  • Using SmartDashboard for telemetry
  • Understanding units (rotations, RPM, etc.)
  • Basic data logging concepts

Success Criteria

  • All telemetry displays correctly on SmartDashboard
  • Encoder resets properly when button is pressed
  • Data updates in real-time (20ms loop)
  • Clean, readable SmartDashboard layout

Challenge 3: PID Position Control

Difficulty: ‚≠ê‚≠ê‚≠ê Intermediate

Objective

Implement PID control to accurately move a motor to specific positions. This builds directly on the 2018 Arm Project concept.

Requirements

1. Create a PID controller for position control of a NEO motor 2. Implement setpoint control using D-pad inputs: - D-pad Up: Move to 0 rotations (home position) - D-pad Right: Move to 5 rotations - D-pad Down: Move to 10 rotations - D-pad Left: Move to -5 rotations 3. Tune PID constants (kP, kD, kI) for smooth motion 4. Implement the following safety features: - Soft limits at ¬±15 rotations - Maximum velocity limit - Current limiting (40A)

Advanced Requirements

  • Display on SmartDashboard:

- Current position vs. setpoint - PID error value - PID output value - Is at setpoint? (within tolerance)

  • Allow tuning PID values through SmartDashboard while robot is running
  • Create a graph showing position over time using NetworkTables

Math Challenge

Create a function that converts arm angle (degrees) to motor rotations, accounting for:

  • Gear ratio (assume 100:1)
  • Starting position offset

Learning Goals

  • Understanding PID control theory
  • Tuning PID controllers
  • Working with setpoints and tolerances
  • Motion profiling basics
  • Safety and limit implementation

Success Criteria

  • Motor reaches all setpoints accurately (¬±0.1 rotations)
  • Motion is smooth without excessive oscillation
  • No overshooting beyond 0.5 rotations
  • All safety limits function correctly
  • PID values can be tuned via SmartDashboard

Reference

Use the 2018 Arm Project document as a guide. Consider using Team Resistance's PIDlab tool for simulation before deploying to hardware.

Challenge 4: Subsystem Architecture

Difficulty: ‚≠ê‚≠ê‚≠ê Intermediate

Objective

Refactor your PID motor control into a proper Command-Based subsystem architecture.

Requirements

1. Create an "Elevator" subsystem class that: - Encapsulates all motor and sensor initialization - Provides methods: setPosition(), getCurrentPosition(), isAtSetpoint() - Implements periodic() for continuous updates - Has proper default command behavior 2. Create Commands for: - MoveToPosition - moves to a specific setpoint - ManualControl - allows manual joystick control - HomeElevator - zeros the elevator to bottom position - HoldPosition - actively holds current position 3. Bind commands to controller buttons properly 4. Implement proper command scheduling and interruption

Advanced Requirements

  • Add state machine with states: IDLE, MOVING, HOLDING, HOMING
  • Create a SmartElevator command that:

- Automatically chooses fastest path to setpoint - Implements motion profiling for smooth acceleration/deceleration

  • Add LED indicators for elevator state (use simulated LEDs on SmartDashboard)

Learning Goals

  • Command-based programming architecture
  • Subsystem design patterns
  • Command scheduling and lifecycle
  • State machine implementation
  • Separation of concerns in code

Success Criteria

  • Clean separation between subsystem and commands
  • Commands can be triggered from multiple sources
  • Commands properly interrupt each other when needed
  • Code follows WPILib best practices
  • All functionality from Challenge 3 still works

Challenge 5: Autonomous Routine

Difficulty: ‚≠ê‚≠ê‚≠ê‚≠ê Advanced

Objective

Create a timed autonomous routine that sequences multiple operations.

Requirements

1. Create an autonomous command that: - Moves elevator to position A (5 rotations) - Waits 1 second - Moves to position B (10 rotations) - Waits 1 second - Returns to home (0 rotations) 2. Use CommandGroups (Sequential, Parallel, Deadline) appropriately 3. Add autonomous selector on SmartDashboard to choose between: - "Safe Auto" - slower, conservative movements - "Fast Auto" - faster movements with motion profiling - "Test Auto" - small movements for testing 4. Implement autonomous with proper timeout safety

Advanced Requirements

  • Add a "pause auto" button that holds current position
  • Create a visual timeline on SmartDashboard showing auto progress
  • Implement error handling - if position can't be reached, safely abort
  • Log all autonomous actions with timestamps

Learning Goals

  • Command composition (Sequential, Parallel, Race)
  • Autonomous programming patterns
  • Timeout and safety mechanisms
  • User interface for autonomous selection
  • Error handling and recovery

Success Criteria

  • Autonomous completes full sequence reliably
  • All three auto modes work correctly
  • Proper timeout handling
  • Clean start/stop behavior
  • Comprehensive logging

Challenge 6: Integration Project

Difficulty: ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê Advanced

Objective

Integrate your elevator subsystem with another mechanism to create a coordinated system.

Requirements

1. Add a second subsystem (choose one): - Option A: Intake with roller wheels - Option B: Simple wrist/pivot mechanism - Option C: Horizontal slider (simulated linear mechanism) 2. Create coordinated commands that control both subsystems: - Example: ScorePosition - moves elevator to height AND positions intake - Example: StowPosition - safely retracts both mechanisms 3. Implement safety interlocks: - Elevator can't move down if intake is deployed - Intake can't deploy if elevator is at bottom - etc. (define safe operating zones) 4. Create a state-based control system where robot remembers its "mode": - STOWED, INTAKING, SCORING_LOW, SCORING_HIGH, etc.

Advanced Requirements

  • Implement collision avoidance between mechanisms
  • Create smart transitions that automatically sequence movements
  • Add "one-button" macros for common operations
  • Implement a visual robot state display on SmartDashboard
  • Add unit tests for your subsystem logic

Learning Goals

  • Multi-subsystem coordination
  • Complex state machines
  • Safety and interlocks
  • System integration
  • Testing strategies

Success Criteria

  • Both subsystems work reliably together
  • No unsafe state combinations possible
  • Smooth transitions between states
  • Comprehensive safety checks
  • Well-documented code with comments

Assessment & Progression

Skill Levels

After completing challenges, programmers should be categorized as:

Level 1 (Challenges 1-2): Basic Controls
  • Can work on simple subsystems
  • Needs guidance on architecture
  • Can read and modify existing code
Level 2 (Challenges 3-4): PID & Subsystems
  • Can implement PID controllers
  • Understands command-based architecture
  • Can design simple subsystems independently
Level 3 (Challenges 5-6): Integration & Autonomy
  • Can create complex autonomous routines
  • Can coordinate multiple subsystems
  • Can mentor Level 1 programmers
Level 4 (Bonus challenges): Advanced Topics
  • Can work on drive code and advanced features
  • Can contribute to team frameworks
  • Can lead programming projects

Code Review Process

All completed challenges must go through code review:

1. Self-Review Checklist: - [ ] Code compiles without warnings - [ ] All functionality requirements met - [ ] Code follows team style guide - [ ] Meaningful variable and method names - [ ] Comments explain "why" not just "what" - [ ] No hardcoded values (use constants) - [ ] Tested on actual hardware (where applicable)

2. Mentor Review: - Mentor will review code with programmer - Discuss improvements and best practices - Identify learning opportunities - Approve or request changes

3. Commit to Team Repository: - Create a branch: training/-challenge - Commit with meaningful messages - Create pull request for review - Respond to feedback - Merge when approved

---

Tips for Success

1. Start Simple: Don't skip ahead. Each challenge builds on the previous one.

2. Test Incrementally: Don't write 100 lines then test. Write 10 lines, test, repeat.

3. Use Version Control: Commit working code frequently. Use branches.

4. Ask Questions: Your mentor is there to help! Don't struggle alone for hours.

5. Read Team Code: Study how experienced team members solve similar problems.

6. Safety First: Always implement safety limits before testing on hardware.

7. Document Your Work: Write README files explaining your approach and what you learned.

8. Pair Programming: Work alongside your mentor or another programmer whenever possible.

9. Use Simulation: Test logic in simulation before deploying to robot.

10. Learn from Failures: Robot broke? Code crashed? These are learning opportunities!
