PID Control



What is PID?

Proportional + Integral + Derivative = Smooth, accurate control




Error = Setpoint - Current Position



Output = (kP √ó Error) + (kI √ó ‚à´Error) + (kD √ó dError/dt)



What Each Term Does




| Term | What It Does | When To Use |
|------|--------------|-------------|
| P (Proportional) | Push harder when farther away | Always start here |
| I (Integral) | Fix steady-state error | Only if needed (careful!) |
| D (Derivative) | Slow down as you approach | Reduces oscillation |


PID Tuning Process




1. Set I and D to 0
2. Increase P until oscillation starts
3. Reduce P by 30-50%
4. Add D to reduce oscillation
5. Add small I only if steady-state error exists
6. Test at different setpoints



Common PID Problems




| Symptom | Likely Cause | Fix |
|---------|--------------|-----|
| Doesn't move | P too small | Increase kP |
| Oscillates | P too large | Decrease kP, increase kD |
| Overshoots | Not enough damping | Increase kD |
| Steady error | No integral term | Add small kI (0.0001 start) |
| Slow/sluggish | P too conservative | Increase kP |

Key Concepts


Subsystem = Hardware Owner

    

  • Owns motors, sensors, etc.
    • 

  • Provides methods to control hardware
    • 

  • Has a default command
    • 

  • Example: ElevatorSubsystem
    • 



Command = Behavior

    

  • Uses subsystem methods
    • 

  • Has lifecycle: initialize ‚Üí execute ‚Üí end
    • 

  • Can be interrupted
    • 

  • Example: MoveToPosition
    • 




Command Structure

java


public class MyCommand extends Command {



public MyCommand(MySubsystem subsystem) {
addRequirements(subsystem);  // Claims the subsystem
}



@Override
public void initialize() {
// Runs ONCE when command starts
}



@Override
public void execute() {
// Runs REPEATEDLY while command runs
}



@Override
public void end(boolean interrupted) {
// Runs ONCE when command ends
}



@Override
public boolean isFinished() {
// Return true to end command
return false;
}
}



Command Compositions


java


// Sequential: One after another
new SequentialCommandGroup(
new MoveToPositionA(),
new WaitCommand(1.0),
new MoveToPositionB()
);



// Parallel: At the same time
new ParallelCommandGroup(
new MoveElevator(),
new MoveIntake()
);



// Race: First to finish ends all
new ParallelRaceGroup(
new MoveUntilDone(),
new WaitCommand(3.0)  // 3 second timeout
);

Debugging Checklist



Robot Won't Connect


    

  • [ ] Is robot powered on?
    • 

  • [ ] Is radio powered on (blue/green lights)?
    • 

  • [ ] Is computer on robot WiFi network?
    • 

  • [ ] Is team number correct in Driver Station?
    • 




Code Won't Deploy


    

  • [ ] Does code compile without errors?
    • 

  • [ ] Is robot connected?
    • 

  • [ ] Is correct team number set (Team 86)?
    • 

  • [ ] Try: Clean rebuild (Ctrl+Shift+P ‚Üí "Clean")
    • 




Motor Won't Move


    

  • [ ] Is motor powered?
    • 

  • [ ] Is CAN ID correct?
    • 

  • [ ] Is motor inverted? (try negative speed)
    • 

  • [ ] Are you in correct mode (teleop/auto)?
    • 

  • [ ] Is motor controller configured?
    • 




Controller Not Working


    

  • [ ] Is controller plugged in?
    • 

  • [ ] Does Driver Station show controller?
    • 

  • [ ] Is controller port correct (usually 0)?
    • 

  • [ ] Try different USB port
    • 




Sensor Reads Wrong Values


    

  • [ ] Is sensor plugged in?
    • 

  • [ ] Is sensor ID correct?
    • 

  • [ ] Check units (rotations vs degrees vs ticks)
    • 

  • [ ] Is sensor reversed? (negative sign)
    • 

  • [ ] Did you zero the sensor?
    •
